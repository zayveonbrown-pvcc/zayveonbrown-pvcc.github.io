    Operating System Terminology

    Operating Systems software packages that coordinate a computer’s internal activities as well as oversee its communication with the outside world.
    Job Queue A storage organization in which objects are ordered in first-in, first-out fashion.
    FIFO Stands for first-in, first-out (abbreviated FIFO and pronounced “FI-foe”).
    Interactive processing Allows a program being executed to carry on a dialogue with the user through remote terminals
    Time-sharing Operating systems that provided service to multiple users at the same time
    Multiprogramming Time is divided into intervals and then the execution of each job is restricted to only one interval at a time
    Application Software Programs for performing tasks particular to the machine’s utilization
    System Software provides the infrastructure that the application software requires, in much the same manner as a nation’s infrastructure (government, roads, utilities, financial institutions, etc.) provides the foundation on which its citizens rely for their individual lifestyles.
    Utility Software Consists of programs for performing activities that are fundamental to computer installations but not included in the operating system
    Graphical User Interface Allows objects to be manipulated, such as files and programs, are represented pictorially on the display as icons
    Shell Older user interfaces, that communicated with users through textual messages using a keyboard and monitor screen
    Kernel contains those software components that perform the very basic functions required by the computer installation
    Device Driver The software units that communicate with the controllers (or at times, directly with peripheral devices) to carry out operations on the peripheral devices attached to the machine
    File Manager Job is to coordinate the use of the machine’s mass storage facilities
    Memory Manager Charged with the task of coordinating the machine’s use of main memory
    Virtual Memory Large “fictional” memory space created by paging
    Boot Strapping Performed by a computer each time it is turned on
    ROM Known as read-only memory
    Deadlock The condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another
    Spooling The technique of holding data for output at a later but more convenient time
    Super user Person gains highly privileged access to the operating system by identifying him- or herself as the administrator (usually by name and password) during the login procedure.
    Multi-factor Authentification An authentication method that requires the user to provide two or more verification factors to gain access to a resource
