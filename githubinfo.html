All About GitHub

Q. What is a version control system?

A. It tracks changes to a file or set of files over time.

Q. What is a Git?

A. A distributed version control system that tracks changes in any set of computer files.

Q. What is GitHub?

A. An online platform that helps developers manage and share code, enabling version control,collaboration, and project hosting.

Q. What is the differnce between Git and GitHub?

A. Git is the tool, and GitHub is the online service built around it.

Q. Who started GitHub and how was it started?

A. Github was developed by Tom Peston-Werner, Chris Wanstrath, Scott Chacon, and PJ Hyett in 2008. They aimed to create a user-friendly hub for hosting, sharing code, and facilitating efficient versoin control.

Q. What company owns it now?

A. Owned by Microsoft.

Git/GitHub terms:
    Respository:
  • A respository is a storage space where files and their version history are kept, crucial for collaborative software development and version control.
    • Commit:
  • The act of saving changes to respository.
    • Fork:
  • "Fork" a respository means to create a personal copy of someone else's project.

Push:
  • Refers to the action of sending committed changes from an local respository to a remote respository.
    • Pull requests:
  • Let you tell other about changes you've pushed to a branch in a respository.
    • Workflows:
  • A configurable automated process made up of one or more jobs.
    • Issues:
  • Items you can create in a respository to plan, discuss and track work.
    • Raw button:
  • Allows you to view or copy the raw content of a file without any styling.
    • Blame button:
  • Used to examine the contents of a file line by line and see when each line was last modified and who the author of the modifications was.
